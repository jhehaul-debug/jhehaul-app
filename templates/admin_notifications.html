🔍 Suppression & Deliverability Check

Check whether an email address is on SendGrid's bounce, block, spam, or unsubscribe lists. If an address is suppressed, SendGrid silently discards mail to it even though logs show "sent."

📋 Why would "sent" emails not arrive in Gmail?

1. Check the spam/junk folder — Gmail often filters transactional mail there. Mark it "Not Spam" to train Gmail.

2. Domain authentication not complete — If the 3 CNAME records in SendGrid aren't verified yet, emails arrive unauthenticated and Gmail is very likely to spam them. In SendGrid → Settings → Sender Authentication, all 3 checks must show green.

3. Suppression list — Use the checker above. If the address appears on a bounce/block/spam list, remove it by clicking the address in SendGrid → Suppressions.

4. "From" email not matching verified sender — The SENDGRID_FROM_EMAIL env var must exactly match the verified sender in SendGrid. Currently: {{ from_email }}

5. SendGrid activity feed — In SendGrid → Activity, search for the recipient email. You can see exactly whether the email was Delivered, Bounced, Blocked, Deferred, or Opened.
